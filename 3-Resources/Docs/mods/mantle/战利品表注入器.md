# Loot Injectors 战利品表注入器

>最后更新：2025-10-18 15:32

Loot Injectors 是由 Mantle 在 1.19.2+ 版本中添加的一种 JSON 格式，它允许在不修改原始 JSON 的情况下向战利品表添加新条目。这提供了与其他执行相同类型修改的模组更好的兼容性。

## Injector Format  注入器格式

战利品注入器 JSON 定义在 `data/<domain>/mantle/loot_injectors/<name>.json` 下，尽管名称 `<name>` 是任意的。它们具有以下格式：

```json
{
    "conditions":[],//必须通过的条件列表,遵循forge/原版格式
    "name":"minecraft:chests/spawn_bonus_chest",//资源位置 修改 loot 表的完整路径，相当于 data/<domain>/loot_tables/ 并去除 .json 扩展名。
    "pools":[
        {
            "name":"main",//根据 Forge 规范的池名称，由模组添加的掉落池应在 JSON 中指定名称。对于未指定名称的表格，第一个池的名称为 main 。未指定名称的每个后续池的名称为 pool# ，其中 # 为池索引（如果包含 main ，则从 0 开始）。例如，掉落表中的第二个池的名称为 pool1 
            "entries":[//池中内容
                {
                    //原版格式的战利品内容
                }
            ]
        }
    ]
}
```

虽然 loot 注入器 `<domain>:<name>` 必须唯一，但如果在相同命名空间下放置另一个注入器，它将根据数据包顺序进行覆盖。使用空的 JSON 覆盖 loot 注入器会导致其被跳过，从而方便地移除注入器。

即数据包A定义了`example:aaa`注入器，注入了某个战利品表，那么优先级更高的数据包B可以通过更改`example":aaa`注入器进行更改原先注入的内容

具有相同 loot 表名称的多个注入器将被合并，应用它们各自的池
